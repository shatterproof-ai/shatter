<h2>Complete Setup Guide</h2>
<p>The infrastructure layer for your thinking. One database, one AI gateway, one chat channel. Any AI you use can plug in. No middleware, no SaaS chains, no Zapier.</p>
<p>This isn&#x27;t a notes app. It&#x27;s a database with vector search and an open protocol — built so that every AI tool you use shares the same persistent memory of you. Claude, ChatGPT, Cursor, Claude Code, whatever ships next month. One brain. All of them.</p>
<p><strong>Before you start:</strong> If you hit a wall during setup, you&#x27;re not on your own. We built a <a href="https://promptkit.natebjones.com/20260224_uq1_guide_02">FAQ</a> that covers the most common questions and gotchas people run into. And if you need real-time help, we created dedicated AI assistants that know this system inside and out, one for each major platform: a <a href="https://www.notion.so/product-templates/Open-Brain-Companion-Claude-Skill-31a5a2ccb526802797caeb37df3ba3cb?source=copy_link">Claude Skill</a>, a <a href="https://chatgpt.com/g/g-69a892b6a7708191b00e48ff655d5597-nate-jones-open-brain-assistant">ChatGPT Custom GPT</a>, and a <a href="https://gemini.google.com/gem/1fDsAENjhdku-3RufY7ystbS1Md8MtDCg?usp=sharing">Gemini GEM</a>. They can walk you through any step, troubleshoot connection issues, and answer questions specific to your setup. Use whichever one matches the AI tool you already use.</p>
<hr/>
<h2>What You&#x27;re Building</h2>
<p>A Slack channel where you type a thought — it automatically gets embedded, classified, and stored in your database — you get a confirmation reply showing what was captured. Then an MCP server that lets any AI assistant search your brain by meaning — and write to it directly.</p>
<h2>What You Need</h2>
<p>About 45 minutes and zero coding experience. You&#x27;ll copy and paste everything.</p>
<h3>Services (All Free Tier)</h3>
<ul>
<li><strong>Supabase</strong> — Your database — stores everything</li>
<li><strong>OpenRouter</strong> — Your AI gateway — understands everything</li>
<li><strong>Slack</strong> — Your capture interface — where you type thoughts</li>
</ul>
<h3>If You Get Stuck</h3>
<p>Follow this guide step by step — it&#x27;s designed to get you through without outside help. But if something goes sideways, Supabase has a free built-in AI assistant in every project dashboard. Look for the chat icon in the bottom-right corner. It has access to all of Supabase&#x27;s documentation and can help with every Supabase-specific step in this guide.</p>
<p>Things it&#x27;s good at:</p>
<ul>
<li>Walking you through where to click when you can&#x27;t find something in the dashboard</li>
<li>Fixing SQL errors if you paste in the error message</li>
<li>Explaining terminal commands and what their output means</li>
<li>Interpreting Edge Function logs when something isn&#x27;t working</li>
<li>Explaining Supabase concepts in plain English (what&#x27;s a Secret key / service role key? what does Row Level Security do?)</li>
</ul>
<p>It can&#x27;t see your screen or run commands for you, but if you paste what you&#x27;re seeing, it can tell you what to do next.</p>
<h2>Two Parts</h2>
<p><strong>Part 1 — Capture</strong> (Steps 1–9): Slack → Edge Function → Supabase. Type a thought, it gets embedded and classified automatically.</p>
<p><strong>Part 2 — Retrieval</strong> (Steps 10–13): Hosted MCP Server → Any AI. Connect Claude, ChatGPT, or any MCP client to your brain with a URL. Read and write from any tool.</p>
<h3>After You&#x27;re Done</h3>
<p>This guide builds the system. The companion prompt pack — <strong><a href="https://promptkit.natebjones.com/20260224_uq1_promptkit_1">Open Brain: Companion Prompts</a></strong> — makes it useful. It includes prompts for migrating your existing AI memories into the brain, migrating an existing second brain system, discovering use cases specific to your workflow, capture templates that optimize metadata extraction, and a weekly review ritual. Finish the setup first, then grab the prompts.</p>

<h1>Part 1 — Capture</h1>
<h2>Step 1: Create Your Supabase Project</h2>
<p>Supabase is your database. It stores your thoughts as raw text, vector embeddings, and structured metadata. It also gives you a REST API automatically.</p>
<ol>
<li>Go to supabase.com and sign up (GitHub login is fastest)</li>
<li>Click <strong>New Project</strong> in the dashboard</li>
<li>Pick your organization (default is fine)</li>
<li>Set Project name: <code class="rounded bg-(--color-code-bg) px-1.5 py-0.5 text-sm font-mono">open-brain</code> (or whatever you want)</li>
<li>Generate a strong Database password — paste into credential tracker NOW</li>
<li>Pick the Region closest to you</li>
<li>Click <strong>Create new project</strong> and wait 1–2 minutes</li>
</ol>
<blockquote>
<p>💡 Grab your Project ref — it&#x27;s the random string in your dashboard URL: <code class="rounded bg-(--color-code-bg) px-1.5 py-0.5 text-sm font-mono">supabase.com/dashboard/project/THIS_PART</code>. Paste it into the tracker.</p>
</blockquote>

<h2>Step 3: Save Your Connection Details</h2>
<p>In the left sidebar: <strong>Settings</strong> (gear icon) → <strong>API</strong>. Copy these into your credential tracker:</p>

<h2>Step 4: Get an OpenRouter API Key</h2>
<p>OpenRouter is a universal AI API gateway — one account gives you access to every major model. We&#x27;re using it for embeddings and lightweight LLM metadata extraction.</p>
<p>Why OpenRouter instead of OpenAI directly? One account, one key, one billing relationship — and it future-proofs you for Claude, Gemini, or any other model later.</p>
<ol>
<li>Go to openrouter.ai and sign up</li>
<li>Go to openrouter.ai/keys</li>
<li>Click <strong>Create Key</strong>, name it <code class="rounded bg-(--color-code-bg) px-1.5 py-0.5 text-sm font-mono">open-brain</code></li>
<li>Copy the key into your credential tracker immediately</li>
<li>Add $5 in credits under Credits (lasts months)</li>
</ol>

<h2>Step 5: Create Your Slack Capture Channel</h2>
<ol>

<h2>Step 6: Create the Slack App</h2>
<p>This is the bridge between Slack and your database.</p>
<h3>Create the App</h3>
<ol>
<li>Go to api.slack.com/apps → <strong>Create New App</strong> → <strong>From scratch</strong></li>
<li>App Name: &quot;Open Brain&quot;, select your workspace</li>
<li>Click <strong>Create App</strong></li>
</ol>
<h3>Set Permissions</h3>
<ol>
<li>Left sidebar → <strong>OAuth &amp; Permissions</strong></li>
<li>Scroll to <strong>Scopes → Bot Token Scopes</strong></li>
<li>Add: <code class="rounded bg-(--color-code-bg) px-1.5 py-0.5 text-sm font-mono">channels:history</code>, <code class="rounded bg-(--color-code-bg) px-1.5 py-0.5 text-sm font-mono">groups:history</code>, <code class="rounded bg-(--color-code-bg) px-1.5 py-0.5 text-sm font-mono">chat:write</code></li>
<li>Scroll up → <strong>Install to Workspace</strong> → Allow</li>
<li>Copy the <strong>Bot User OAuth Token</strong> (starts with <code class="rounded bg-(--color-code-bg) px-1.5 py-0.5 text-sm font-mono">xoxb-</code>) into credential tracker</li>
</ol>
<h3>Add App to Channel</h3>
<p>In Slack, open your capture channel and type: <code class="rounded bg-(--color-code-bg) px-1.5 py-0.5 text-sm font-mono">/invite @Open Brain</code></p>
<blockquote>
<p>💡 Don&#x27;t set up Event Subscriptions yet — you need the Edge Function URL first (Step 7).</p>
</blockquote>

<h2>Step 7: Deploy the Edge Function</h2>
<p>This is the brains of the operation. One function receives messages from Slack, generates an embedding, extracts metadata, stores everything in Supabase, and replies with a confirmation.</p>
<blockquote>
<p>💡 <strong>New to the terminal?</strong> The &quot;terminal&quot; is the text-based command line on your computer. On Mac, open the app called <strong>Terminal</strong> (search for it in Spotlight). On Windows, open <strong>PowerShell</strong>. Everything below gets typed there, not in your browser.</p>
</blockquote>
<h3>Install the Supabase CLI</h3>
<blockquote>
<p>💡 <strong>Mac users:</strong> If you already have Homebrew installed (you&#x27;ll know — it&#x27;s the thing you install with <code class="rounded bg-(--color-code-bg) px-1.5 py-0.5 text-sm font-mono">brew</code>), use the first option. <strong>Windows users:</strong> use Scoop — Supabase recommends it over npm for Windows because it handles PATH and permissions cleanly. <strong>Linux or Mac without Homebrew:</strong> use npm.</p>
</blockquote>
<pre class="whitespace-pre-wrap break-words rounded-lg bg-(--color-code-bg) p-4 text-sm font-mono leading-snug text-neutral-900 dark:text-neutral-200"><code># Mac with Homebrew
brew install supabase/tap/supabase
</code></pre>
<pre class="whitespace-pre-wrap break-words rounded-lg bg-(--color-code-bg) p-4 text-sm font-mono leading-snug text-neutral-900 dark:text-neutral-200"><code># Windows with Scoop (recommended)
# Install Scoop first if you don&#x27;t have it:
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
Invoke-RestMethod -Uri https://get.scoop.sh | Invoke-Expression

# Then install Supabase:
scoop bucket add supabase https://github.com/supabase/scoop-bucket.git
scoop install supabase
</code></pre>
<pre class="whitespace-pre-wrap break-words rounded-lg bg-(--color-code-bg) p-4 text-sm font-mono leading-snug text-neutral-900 dark:text-neutral-200"><code># Linux or Mac without Homebrew
npm install -g supabase
</code></pre>
<p>Verify it worked:</p>

<h2>Step 8: Connect Slack to the Edge Function</h2>
<ol>
<li>Go to api.slack.com/apps → select your Open Brain app</li>
<li>Left sidebar → <strong>Event Subscriptions</strong> → toggle <strong>Enable Events ON</strong></li>

<li>Under <strong>Subscribe to bot events</strong>, add both: <code class="rounded bg-(--color-code-bg) px-1.5 py-0.5 text-sm font-mono">message.channels</code> and <code class="rounded bg-(--color-code-bg) px-1.5 py-0.5 text-sm font-mono">message.groups</code></li>
<li>Click <strong>Save Changes</strong> (reinstall if prompted)</li>
</ol>
<blockquote>
<p>⚠️ <strong>You need both events.</strong> Slack treats public and private channels as separate entity types. Public channels fire <code class="rounded bg-(--color-code-bg) px-1.5 py-0.5 text-sm font-mono">message.channels</code>, private channels fire <code class="rounded bg-(--color-code-bg) px-1.5 py-0.5 text-sm font-mono">message.groups</code>. If you only add one, messages in the other channel type will silently fail — no error, just nothing happens. Add both so you&#x27;re covered regardless of how your capture channel is configured.</p>

<h1>Part 2 — Retrieval</h1>
<h2>A Quick Note on Architecture</h2>
<p>MCP servers can run two ways: locally on your computer, or hosted in the cloud.</p>
<p>The local approach means installing Node.js, building a TypeScript project, and running a server process on your machine. Every AI client you connect needs the full path to that server plus your database credentials pasted into a config file. If your laptop is closed, your brain is offline. If you switch computers, you set it up again.</p>
<p>We&#x27;re not doing that.</p>
<p>Your capture system already runs on Supabase — the Edge Function you deployed in Part 1 handles Slack messages without anything running on your computer. The MCP server works the same way. One more Edge Function, deployed to the same project, reachable from anywhere. Your AI clients connect with a URL. No build steps, no local dependencies, no credentials on your machine.</p>
<p>If you want to run locally — maybe you&#x27;re a developer who prefers that, or you want to customize beyond what Edge Functions allow — the MCP TypeScript SDK with StdioServerTransport works great. The <a href="https://supabase.com/docs/guides/getting-started/byo-mcp">Supabase docs on deploying MCP servers</a> cover both approaches. Everything below uses hosted.</p>
<hr/>
<h2>Step 10: Create an Access Key</h2>
<p>Your MCP server will be a public URL. The Supabase project ref in that URL is random enough that nobody will stumble onto it, but let&#x27;s close the gap entirely. You&#x27;ll generate a simple access key that the server checks on every request. Takes 30 seconds.</p>
<p>In your terminal, generate a random key:</p>
<pre class="whitespace-pre-wrap break-words rounded-lg bg-(--color-code-bg) p-4 text-sm font-mono leading-snug text-neutral-900 dark:text-neutral-200"><code># Mac/Linux
openssl rand -hex 32
</code></pre>
<pre class="whitespace-pre-wrap break-words rounded-lg bg-(--color-code-bg) p-4 text-sm font-mono leading-snug text-neutral-900 dark:text-neutral-200"><code># Windows (PowerShell)
-join ((1..32) | ForEach-Object { &#x27;{0:x2}&#x27; -f (Get-Random -Maximum 256) })
</code></pre>
<p>Copy the output — it&#x27;ll look something like <code class="rounded bg-(--color-code-bg) px-1.5 py-0.5 text-sm font-mono">a3f8b2c1d4e5...</code> (64 characters). Paste it into your credential tracker under MCP Access Key.</p>
<p>Set it as a Supabase secret:</p>
<pre class="whitespace-pre-wrap break-words rounded-lg bg-(--color-code-bg) p-4 text-sm font-mono leading-snug text-neutral-900 dark:text-neutral-200"><code>supabase secrets set MCP_ACCESS_KEY=your-generated-key-here
</code></pre>

<h1>Troubleshooting</h1>
<p>If the specific suggestions below don&#x27;t solve your issue, remember: the Supabase AI assistant (chat icon, bottom-right of your dashboard) can help diagnose problems with anything Supabase-related. Paste the error message and tell it what step you&#x27;re on.</p>
<h2>Capture Issues (Part 1)</h2>
<h3>Slack says &quot;Request URL not verified&quot;</h3>
<p>Your Edge Function isn&#x27;t deployed or isn&#x27;t reachable. Run the deploy command again and check the output for errors.</p>
<pre class="whitespace-pre-wrap break-words rounded-lg bg-(--color-code-bg) p-4 text-sm font-mono leading-snug text-neutral-900 dark:text-neutral-200"><code>supabase functions deploy ingest-thought --no-verify-jwt
</code></pre>
<h3>Messages aren&#x27;t triggering the function</h3>
<p>Check Event Subscriptions — make sure both <code class="rounded bg-(--color-code-bg) px-1.5 py-0.5 text-sm font-mono">message.channels</code> and <code class="rounded bg-(--color-code-bg) px-1.5 py-0.5 text-sm font-mono">message.groups</code> are listed (public channels use the first, private channels use the second — you need both). Verify the app is invited to the channel. Confirm the channel ID in your secrets matches the actual channel.</p>
<h3>Slack creates duplicate database entries</h3>
<p>Slack retries webhook delivery if it doesn&#x27;t get a response within 3 seconds. If your Edge Function takes longer than that (embedding + metadata extraction can take 4-5 seconds), Slack sends the event again, and you get two rows. This is a known edge case. The captures are identical, so it doesn&#x27;t affect search — but if it bothers you, you can delete the duplicate row in the Supabase Table Editor.</p>
<h3>Function runs but nothing in the database</h3>
<p>Check Edge Function logs: Supabase dashboard → Edge Functions → ingest-thought → Logs. Most likely the OpenRouter key is wrong or has no credits.</p>
<pre class="whitespace-pre-wrap break-words rounded-lg bg-(--color-code-bg) p-4 text-sm font-mono leading-snug text-neutral-900 dark:text-neutral-200"><code>supabase secrets list
</code></pre>
<h3>No confirmation reply in Slack</h3>
<p>The bot token might be wrong, or <code class="rounded bg-(--color-code-bg) px-1.5 py-0.5 text-sm font-mono">chat:write</code> scope wasn&#x27;t added. Go to your Slack app → OAuth &amp; Permissions and verify. If you added the scope after installing, you need to reinstall the app.</p>
<h3>Metadata extraction seems off</h3>
<p>That&#x27;s normal — the LLM is making its best guess with limited context. The metadata is a convenience layer on top of semantic search, not the primary retrieval mechanism. The embedding handles fuzzy matching regardless.</p>
<h2>Retrieval Issues (Part 2)</h2>
<h3>Claude Desktop tools don&#x27;t appear</h3>
<p>Make sure you added the connector in Settings → Connectors (not by editing the JSON config file). Verify the connector is enabled for your conversation — click the &quot;+&quot; button at the bottom of the chat, then Connectors, and check that Open Brain is toggled on. If the connector was added but tools still don&#x27;t show, try removing and re-adding it with the same URL.</p>
<h3>ChatGPT doesn&#x27;t use the Open Brain tools</h3>
<p>First, confirm Developer Mode is enabled (Settings → Apps &amp; Connectors → Advanced settings). Without it, ChatGPT only exposes limited MCP functionality that won&#x27;t cover Open Brain&#x27;s full toolset. Next, check that the connector is active for your current conversation — look for it in the tools/apps panel. If it&#x27;s connected but ChatGPT ignores it, be direct: &quot;Use the Open Brain search_thoughts tool to search for [topic].&quot; ChatGPT often needs explicit tool references the first few times before it starts picking them up automatically.</p>
<h3>Getting 401 errors</h3>
<p>The access key doesn&#x27;t match what&#x27;s stored in Supabase secrets. Double-check that the <code class="rounded bg-(--color-code-bg) px-1.5 py-0.5 text-sm font-mono">?key=</code> value in your URL matches your MCP Access Key exactly. If you&#x27;re using the header approach (Claude Code or mcp-remote), the header must be <code class="rounded bg-(--color-code-bg) px-1.5 py-0.5 text-sm font-mono">x-brain-key</code> (lowercase, with the dash).</p>
<h3>Search returns no results</h3>
<p>Make sure you sent test messages in Part 1 first. Try asking the AI to &quot;search with threshold 0.3&quot; for a wider net. If that still returns nothing, check the Edge Function logs in the Supabase dashboard for errors.</p>
<h3>Tools work but responses are slow</h3>
<p>First search on a cold function takes a few seconds — the Edge Function is waking up. Subsequent calls are faster. If it&#x27;s consistently slow, check your Supabase project region — pick the one closest to you.</p>
<h3>Capture tool saves but metadata is wrong</h3>
<p>Same as Slack capture — the metadata extraction is best-effort. The embedding is what powers semantic search, and that works regardless of how the metadata gets classified. If you consistently want a specific classification, use the capture templates from the prompt kit to give the LLM clearer signals.</p>

<h2>How It Works Under the Hood</h2>
<p>When you type a message in Slack: Slack sends it to your Edge Function → the function generates an embedding (1536-dimensional vector of meaning) AND extracts metadata via LLM in parallel → both get stored as a single row in Supabase → the function replies in your Slack thread with a summary.</p>
<p>When you capture from any AI via MCP: your AI client sends the text to the capture_thought tool → the MCP server generates an embedding AND extracts metadata in parallel (same pipeline as Slack) → stored as a single row → confirmation returned to your AI.</p>
<p>When you ask your AI about it: your AI client sends the query to the MCP Edge Function → the function generates an embedding of your question → Supabase matches it against every stored thought by vector similarity → results come back ranked by meaning, not keywords.</p>
<p>The embedding is what makes retrieval powerful. &quot;Sarah&#x27;s thinking about leaving&quot; and &quot;What did I note about career changes?&quot; match semantically even though they share zero keywords. The metadata is a bonus layer for structured filtering on top.</p>
<h3>Swapping Models Later</h3>
<p>Because you&#x27;re using OpenRouter, you can swap models by editing the model strings in the Edge Function code and redeploying. Browse available models at openrouter.ai/models. Just make sure embedding dimensions match (1536 for the current setup).</p>
